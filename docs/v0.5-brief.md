# v0.5 — glossary + roamable web. Brief.

Done when: a newcomer opens a shared link and can roam from a glossary term to the
experiments/datasets that use it, and every page shows where its facts came from.
public/internal split honored. Build on the existing `kb/` (64 real nodes) + the
SQLite index. Respect CLAUDE.md invariants — especially files-are-truth,
provenance-or-nothing, deterministic-first.

## 1. Glossary auto-extract → `concept` nodes

- Source: existing `card` and `page` nodes. Target type `concept` (already in schema).
- Deterministic first. Pull terms already defined in-text: acronym expansions
  ("LLMzSzŁ (…)"), definition patterns ("X to …", "X — …"), legenda/glossary
  sections, `matrix`/axis table headers. This covers most lab jargon.
- Candidate detection: terms appearing as entities/tags or repeated acronyms across
  ≥2 nodes with no `concept` node yet.
- LLM (small MLX) ONLY for mentioned-but-undefined terms. Hard constraints:
  definition MUST be grounded in the nodes that use the term (carry them as
  `provenance`); 1–2 sentences; `status: draft` so a human verifies. If it can't be
  grounded, do NOT fabricate — skip, or drop a `question` node. The provenance
  invariant applies to glossary too.
- Linking: each `concept` links to the nodes that use it ("used in: …"), derived
  from existing `links`/mentions or co-occurrence. This is what makes the roam work
  — without it the glossary is dead-end definitions.

## 2. Roamable web → `bb web:build`

- Render `kb/` → static Hugo site. The Hugo content dir is GENERATED from `kb/` —
  never hand-edited, never written back. Like the index, the site is fully
  rebuildable.
- Wikilinks: Goldmark won't parse `[[id]]`. Preprocess — rewrite `[[ULID]]` → a real
  Hugo link via the id→path/permalink map from the index. Don't hand-roll a parser;
  use the index.
- Backlinks: precompute from the `links` table at build time, inject per page. Not
  at template time.
- Provenance: every node page renders `provenance` as clickable source links (git
  ref → GitHub blob URL at the pinned sha; web ref → URL). This satisfies "every
  rendered claim shows a source link."
- Permalinks: stable, id-based (`/node/<id>/`) so links survive title changes; add a
  human slug for readability.
- MOC pages: one index page per `moc` facet, nodes grouped by `type`, generated from
  the index.
- Graph view: per-node LOCAL/ego graph (this node + typed neighbors), NOT a global
  force-directed hairball. The global graph is a gimmick at scale; the local
  neighborhood is the useful nav aid.
- public/internal: filter by `visibility` at the emit step — only `public` nodes
  reach the public content dir. Emit the internal set to a separate dir but leave it
  stubbed/unsecured (no auth system in v0.5).
- `## State` section: render whatever is already there (Tier A body / experiment
  result+decision). Do NOT synthesize it — that's v1.5.

## 3. Keep it the "prophet" view, not a clone of slayer's site

- The lab already publishes its pages. The value here is the cross-linked,
  provenance-anchored, glossary-indexed VIEW — not republishing. If the output looks
  like a copy of `slayerlabs/slayer`, the point is lost. Lead with glossary, used-in
  backlinks, provenance, cross-source links.

## Out of scope for v0.5 (do not build)

synthesis/rollup (v1.5), auth for the internal build (stub), global graph viz, write
tools / review queue (v1.5), Discord / Tier B (v1).
